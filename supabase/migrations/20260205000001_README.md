# Migration 20260205000001: WhatsApp Intent Privacy-First Storage

**Issue:** #91 - Processar mensagens WhatsApp para timeline
**Date:** 2026-02-05
**Type:** Schema transformation (privacy-critical)

## Quick Summary

This migration transforms WhatsApp message storage from **raw text** to **extracted intents**, ensuring compliance with WhatsApp Terms of Service and LGPD/GDPR data minimization principles.

### Critical Change
- **REMOVED:** `whatsapp_messages.content_text` column (privacy violation)
- **ADDED:** 11 intent-based columns + vector embeddings for semantic search

## What This Migration Does

### 1. Enables pgvector Extension
```sql
CREATE EXTENSION IF NOT EXISTS vector;
```
Required for storing 768-dimensional embeddings from Gemini's `text-embedding-004` model.

### 2. Creates Enums
- `whatsapp_intent_category`: 9 values (question, response, scheduling, document, audio, social, request, update, media)
- `whatsapp_intent_sentiment`: 4 values (positive, neutral, negative, urgent)

### 3. Alters whatsapp_messages Table

**Dropped columns:**
- `content_text` (raw message text - WhatsApp ToS violation)

**Added columns:**
- `intent_summary` TEXT - Human-readable summary (safe for display)
- `intent_category` ENUM - Message classification
- `intent_sentiment` ENUM - Sentiment analysis
- `intent_urgency` SMALLINT (1-5) - Priority score
- `intent_topic` VARCHAR(50) - Main topic
- `intent_action_required` BOOLEAN - Requires user response
- `intent_mentioned_date` DATE - Extracted date
- `intent_mentioned_time` TIME - Extracted time
- `intent_media_type` VARCHAR(20) - Media type if present
- `intent_confidence` DECIMAL(3,2) - AI confidence score
- `intent_embedding` vector(768) - Semantic search vector
- `processing_status` VARCHAR(20) - Pipeline status

### 4. Alters contact_network Table

**Dropped columns:**
- `last_message_preview` (raw text preview)
- `last_message_direction` (incoming/outgoing)

**Added columns:**
- `last_intent_preview` TEXT - Intent summary preview
- `last_intent_category` ENUM - Category of last message
- `last_intent_sentiment` ENUM - Sentiment of last message
- `last_intent_urgency` SMALLINT - Urgency of last message

### 5. Creates Indexes
1. **Vector similarity search:** IVFFlat index on `intent_embedding` (cosine distance)
2. **Intent filtering:** Composite index on (user_id, category, urgency, created_at)
3. **Action required:** Index for task management queries
4. **Scheduled messages:** Index for calendar integration

### 6. Creates Function: search_messages_by_intent()
```sql
search_messages_by_intent(
  _user_id UUID,
  _query_embedding vector(768),
  _limit INTEGER DEFAULT 20,
  _category whatsapp_intent_category DEFAULT NULL,
  _min_urgency SMALLINT DEFAULT 1
)
```
Returns messages ranked by semantic similarity (cosine distance).

### 7. Updates Trigger
- **Old:** `update_contact_last_message_preview()` → Stored raw text
- **New:** `update_contact_last_intent_preview()` → Stores intent summary

Fires when `processing_status = 'completed'` to update contact preview.

### 8. Drops Obsolete Function
- `backfill_contact_message_previews()` → No longer needed

## Performance Impact

### Storage Per Message
- **Before:** ~500 bytes (raw text + metadata)
- **After:** ~3,200 bytes (intent + embedding)
- **Trade-off:** Higher storage cost BUT privacy compliant + semantic search

### Query Performance
- Vector similarity search: ~50ms for 10k messages (IVFFlat index)
- Intent filtering: <10ms (composite B-tree index)
- Action-required queries: <5ms (dedicated index)

### API Costs (Gemini)
- Intent extraction: ~$0.00002/message (gemini-1.5-flash)
- Embedding generation: ~$0.00001/message (text-embedding-004)
- **Total:** ~$0.00003/message = **$0.30 for 10k messages/month**

## Known Issues / Limitations

1. **Existing messages:** If `content_text` had data, it's now lost. Intent must be re-extracted from original source if available.
2. **Backfill needed:** Run intent extraction on existing messages if any.
3. **Embedding cost:** First-time embedding generation for large message history may incur significant API costs.
