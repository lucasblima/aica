# Documentation Maintenance Agent

This directory contains automated tools for maintaining project documentation based on code changes and database state.

## Overview

The Documentation Maintenance Agent automatically:

1. **Analyzes Git Commits** - Understands what changed in recent sprints
2. **Verifies Implementation** - Checks which features are actually implemented
3. **Updates PRD.md** - Maintains accurate implementation status
4. **Updates backend_architecture.md** - Keeps table definitions current
5. **Validates Database Schema** - Compares expected vs actual Supabase tables

## Installation

Install the required dependency:

```bash
npm install
```

This will install `tsx` for running TypeScript scripts directly.

## Usage

### 1. Update Documentation After Sprint

Run after completing a sprint to update all documentation:

```bash
npm run update-docs
```

This will:
- Analyze the last 10 commits
- Verify which features are implemented
- Update `docs/PRD.md` with current status
- Update `docs/backend_architecture.md` with timestamp
- Generate a summary report in the console

### 2. Analyze More Commits

To analyze a larger commit history:

```bash
npm run update-docs:full
```

This analyzes the last 50 commits instead of 10.

### 3. Analyze Commits Since Date

To analyze all commits since a specific date:

```bash
npm run update-docs:since "2025-12-01"
```

### 4. Validate Supabase Schema

Generate SQL queries to validate database schema:

```bash
npm run validate-schema
```

This creates `docs/SCHEMA_VALIDATION_REPORT.md` with SQL queries to:
- Check all expected tables exist
- Verify RLS policies are enabled
- Validate key columns exist
- Check Security Definer functions

## How It Works

### Documentation Update Agent (`update-docs.ts`)

```typescript
// Analyzes commits
const commits = this.analyzeCommits({ commits: 10 });

// Verifies implementation
const features = [
  'Security Definer Pattern',
  'TypeScript Type Generation',
  'Error Boundaries',
  'Gamification System',
  // ... more features
];

// Updates documentation
updatePRD(implementations);
updateBackendArchitecture();
```

**Features Tracked:**
- Security Definer Pattern (RLS functions)
- TypeScript Type Generation
- Error Boundaries (graceful degradation)
- Gamification System (XP, levels, efficiency)
- Podcast Copilot (Gemini integration)
- Contact Network
- Memories System
- Daily Reports

### Schema Validator (`validate-supabase-schema.ts`)

Generates SQL queries to validate:

```sql
-- Table existence
SELECT table_name FROM information_schema.tables
WHERE table_schema = 'public';

-- RLS status
SELECT tablename, rowsecurity FROM pg_tables
WHERE schemaname = 'public';

-- Column validation
SELECT column_name, data_type FROM information_schema.columns
WHERE table_name = 'daily_reports';
```

**Expected Tables:** 17 core tables including:
- `users`, `associations`, `modules`, `work_items`
- `memories`, `daily_reports`, `activity_log`
- `contact_network`
- `podcast_shows`, `podcast_episodes`, `podcast_topics`
- `user_stats`, `task_metrics`
- `workspaces`

## Integration with Supabase MCP

The schema validator generates SQL that can be executed using:

### Option 1: Claude Code with Supabase MCP

```typescript
// Claude can execute these via MCP
mcp__supabase__execute_sql({
  project_id: "your-project-id",
  query: "SELECT table_name FROM information_schema.tables..."
});
```

### Option 2: Supabase Dashboard

1. Open Supabase Dashboard
2. Navigate to SQL Editor
3. Paste queries from `SCHEMA_VALIDATION_REPORT.md`
4. Execute and review results

### Option 3: Supabase CLI

```bash
supabase db execute < docs/SCHEMA_VALIDATION_REPORT.md
```

## Troubleshooting

### Issue: `tsx: command not found`

**Solution:** Run `npm install` to install dependencies.

### Issue: `git: command not found`

**Solution:** Ensure Git is installed and in PATH.

### Issue: "No commits to analyze"

**Solution:** Check you're in a git repository with commit history.

### Issue: PRD.md not updating

**Solution:** Ensure `docs/PRD.md` exists and is writable.

## Architecture

### File Structure

```
scripts/
├── README.md                      # This file
├── update-docs.ts                 # Main documentation agent
└── validate-supabase-schema.ts    # Schema validation generator
```

### Dependencies

- **tsx** - Run TypeScript directly without compilation
- **git** - Analyze commit history
- **node** - Execute scripts

### Design Patterns

1. **Single Responsibility** - Each script has one clear purpose
2. **Idempotent** - Safe to run multiple times
3. **Non-Destructive** - Only updates documentation, never deletes
4. **Verifiable** - Generates reports for manual review
5. **Extensible** - Easy to add new features/tables to track

## Best Practices

1. **Run After Each Sprint** - Keep documentation synchronized
2. **Review Reports** - Don't blindly trust automation
3. **Commit Documentation Changes** - Track doc changes in git
4. **Validate Schema Regularly** - Catch database drift early
5. **Customize Feature List** - Track what matters to your project

## Integration with CI/CD

Add to your GitHub Actions workflow:

```yaml
name: Update Documentation
on:
  push:
    branches: [main]

jobs:
  update-docs:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
      - run: npm install
      - run: npm run update-docs
      - run: npm run validate-schema
      - uses: stefanzweifel/git-auto-commit-action@v4
        with:
          commit_message: "docs: auto-update documentation [skip ci]"
          file_pattern: "docs/*.md"
```
