================================================================================
                        ARCHITECTURE HIGHLIGHTS
================================================================================

KEY DECISION: Props-Based Callbacks
  - No internal navigation (enables FSM in parent)
  - Parent component controls mode transitions
  - Clean separation of concerns
  - Enables Finite State Machine pattern

KEY DESIGN: Episode Caching
  - Lazy load episodes on show expansion
  - Cache in component state
  - Prevent unnecessary re-fetching
  - Smooth user experience

KEY FEATURE: Generic Project Format
  - episodeToProject() converts database to interface
  - Supports future project types (video, article)
  - Metadata field for type-specific data
  - Extensible without code changes

KEY ACHIEVEMENT: Zero Race Conditions
  - Proper state management
  - No useEffect for navigation
  - Explicit transitions
  - FSM-ready architecture
