INTEGRAÇÃO COM WORKSPACE
================================================================================

STATE ACCESSED
  - setup.guestName: Nome do convidado
  - setup.theme: Tema do episódio
  - research.dossier: Dossier gerado (Dossier | null)
  - research.customSources: Fontes customizadas (CustomSource[])
  - research.isGenerating: Loading state (boolean)
  - research.lastGenerated: Timestamp (Date | null)
  - research.error: Mensagem de erro (string | null)

ACTIONS CALLED
  - actions.generateDossier(): Gera dossier inicial
  - actions.regenerateDossier(): Regenera com fontes
  - actions.addCustomSource(source): Adiciona fonte
  - actions.removeCustomSource(sourceId): Remove fonte
  - actions.setStage('pauta'): Navega para próxima stage

AUTO-SAVE INTEGRATION
  - useAutoSave hook: debounce 2s
  - Trigger: Qualquer mudança em customSources
  - Flag: isDirty marca estado não-salvo
  - Ready: Apenas falta completar integração no provider

FLUXO DE USO (USER JOURNEY)
================================================================================

1. ACESSO DO STAGE
   SetupStage (completo)
   → Clica "Próximo: Pesquisa"
   → ResearchStage carregado
   → Guest name e theme já disponíveis

2. GERAR DOSSIER
   ResearchStage vazio
   → Clica "Gerar Dossier"
   → Button disabled, spinner visível
   → API chamada (onGenerateDossier callback)
   → Dossier recebido e exibido
   → Button muda para "Regenerar com Fontes"
   → Tabs Bio/Ficha/Notícias agora visíveis

3. ADICIONAR FONTES
   ResearchStage com dossier
   → Clica "Adicionar Fontes"
   → Modal abre (fixed overlay)
   → Seleciona opção: Texto/URL/Arquivo
   → Preenche campo apropriado
   → Clica "Adicionar"
   → CustomSource adicionada ao state
   → Modal fecha (form limpo)
   → Fonte visível na lista esquerda

4. REGENERAR COM FONTES
   ResearchStage com dossier + fontes
   → Clica "Regenerar com Fontes"
   → customSources incluídas na API call
   → Loading spinner visível
   → Novo dossier recebido com mais contexto
   → Conteúdo atualizado nas tabs

5. CHAT COM AICA
   ResearchStage com dossier
   → Digita pergunta no input chat
   → Pressiona Enter ou clica Send
   → Mensagem usuario adicionada (orange bubble)
   → API chamada (mock ou Gemini Live)
   → Resposta exibida (gray bubble)
   → Scrolling automático para mensagem nova

6. NAVEGAÇÃO
   ResearchStage completo (biography > 200 chars)
   → Clica "Próximo: Pauta"
   → actions.setStage('pauta') chamado
   → PautaStage carregado
   → Research state preservado no context
   → Dossier disponível para geração de pauta
