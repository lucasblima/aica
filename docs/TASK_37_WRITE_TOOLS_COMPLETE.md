**Returns:**
```python
{
    "status": "success",
    "message": "Tarefa atualizada com sucesso!"
}
```

Removes a task from the system.

**Parameters:**
- `task_id` (str) - UUID of task to delete

**Returns:**
```python

    "status": "success",
    "message": "Tarefa removida com sucesso!"
}
```

## Security & Architecture

### RLS Compliance
All write tools respect Row-Level Security policies:
- User ID extracted from JWT-verified session context
- Tools never accept user_id as parameter (prevents privilege escalation)
- `_get_user_id()` helper enforces authentication

### Error Handling
All tools follow consistent error response pattern:
```python
{
    "status": "error",
    "message": "Descriptive error message"
}
```

### Validation
- Input validation for enums (priority, status, etc.)
- Range validation for numeric fields (quadrant 1-4, confidence 0-1)
- Type validation for complex fields (dicts, lists)

- All tools use `.eq("user_id", user_id)` filter first (index hit)

- Bulk operations minimize N+1 queries

## Maintainer Notes

**When adding new write tools:**

1. **Always extract user_id via `_get_user_id(tool_context)`**
   - NEVER accept user_id as a parameter
   - This prevents privilege escalation attacks

2. **Validate all inputs**
   - Enums: Check against valid values
   - Numbers: Check ranges (0-1 for confidence, 1-4 for quadrants)
   - Types: Ensure dicts are dicts, lists are lists

3. **Return consistent response format**
   ```python
   {
       "status": "success" | "error",
       "message": "Human-readable message",
       # ... additional data
   }
   ```

4. **Update agent instruction**
